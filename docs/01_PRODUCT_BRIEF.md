# Product Brief — 패독 코리아 (Paddock Korea)

## 1. Product Summary

이 서비스는 **한국의 F1 팬(특히 2030 신규 팬)**이 분산된 플랫폼에서 정보를 찾아 헤매고 소통 공간이 없어 이탈하는 문제를, **팀 기반 커뮤니티와 입문 가이드를 하나의 앱에서 제공**하는 방식으로 해결하도록 돕는다.

---

## 2. Problem

F1 the Movie(2025) 개봉 이후 한국의 F1 팬이 급격히 증가하였으나, 팬들이 모여 소통할 수 있는 전용 공간이 없다. 현재 팬들은 유튜브, 블로그, 카카오 오픈채팅, 디시인사이드 등 여러 플랫폼을 오가며 단편적으로 정보를 소비하고 있어 커뮤니티의 응집력이 매우 약하다. 또한 F1 특유의 복잡한 용어와 규칙은 신규 팬의 진입 장벽으로 작용해 커뮤니티에 안착하지 못한 채 이탈하는 경우가 많다. F1은 팀 스포츠임에도 불구하고 기존 커뮤니티에는 팀별 소속감을 형성할 수 있는 구조와 기능이 전혀 없어, 팬들이 장기적으로 스포츠에 몰입하기 어렵다.

---

## 3. Target User

- **주요 사용자:** 한국의 20~30대 F1 팬 (특히 F1 영화·넷플릭스를 통해 유입된 신규 팬)
- **사용 상황:** 레이스 위켄드(금~일) 전후 경기 정보 탐색, 경기 중 실시간 반응 공유, 평일 팀·드라이버 관련 콘텐츠 소비
- **사용자의 기술 수준:** 스마트폰 앱에 익숙한 일반 사용자; F1 지식은 입문~중급 수준이 혼재
- **사용자가 기대하는 결과:** 좋아하는 팀 팬끼리 모여 응원하고, 어려운 F1 용어를 쉽게 배우며, 경기를 함께 즐길 수 있는 커뮤니티에 소속감을 느끼는 경험

---

## 4. Core Value

팀 기반 소속감, 입문자를 위한 학습 콘텐츠, 팬들 간의 밈과 응원 문화를 **하나의 공간**에서 제공해 한국 F1 팬이 흩어지지 않고 깊이 몰입할 수 있도록 한다.

---

## 5. MVP Scope (현재 구현됨)

현재 앱은 6개 탭으로 구성된다 — Home / Board / Meme Box / F1 101 / Pit Wall / My Page.

- **팀 선택 및 닉네임 설정 온보딩:** 닉네임과 선호 팀(2026 시즌 11개 팀 또는 "응원 팀 없음"·"올팬")을 설정하고 참여하는 플로우. 실제 팀은 **최대 2개**까지 선택 가능.
- **홈(Home):** 다음/진행 중 그랑프리 현황(D-day·LIVE), 내 팀 응원·랭킹, **전체 실시간 채팅**(컴팩트 + 전체화면)을 한 화면에 모은 대시보드.
- **전체 채팅(Main Straight):** 팀 무관 모든 팬이 소통하는 실시간 채팅. Home 안에 통합되어 있다.
- **게시판(Board):** 전체/팀별 스코프로 글·댓글·좋아요를 남기는 커뮤니티. 팀 게시판 작성 권한은 응원 팀(또는 "올팬")에만 부여되고, 그 외 팀 게시판은 읽기 전용. 최신순·인기순 정렬 지원. (기존 "팀별 채팅(The Garage)"을 게시판 형태로 발전시킨 기능)
- **응원(Cheer):** 하루 1회 내 팀에 응원(+1)을 보내고, 팀별 누적 응원 랭킹을 확인.
- **밈 공유 공간(Meme Box):** 이미지 URL·캡션으로 밈을 게시하고 좋아요로 반응하는 피드.
- **F1 입문 가이드(F1 101):** 주요 용어 사전, 경기 진행 방식, 레이스 위켄드 구조를 카드 형식으로 제공.
- **Pit Wall:** OpenF1 API 기반 드라이버·컨스트럭터 챔피언십 순위, 레이스 위켄드 일정(KST), 세션별 결과·드라이버 헤드샷. 실패 시 정적 폴백.
- **My Page:** 닉네임·응원 팀(최대 2) 변경, 라이트/다크 테마 토글, 내가 작성한 게시글 목록.
- **테마:** 라이트/다크 모드 전환(헤더·My Page).

---

## 6. Non-goals

### 여전히 제외 (이번 단계에서도 안 함)

- 결제 / 굿즈 스토어 (The Paddock Store)
- 다중 외부 API 연동 — 외부 API는 OpenF1 하나로 유지
- 복잡한 관리자 권한 및 모더레이션 시스템
- 라이벌 팀 도발(Banter) 기능

### 고도화 단계에서 해제된 제약 (이제 도입 가능)

> 아래는 초기 MVP에서 제외했으나, 고도화 단계에서 해제되어 도입 가능한 영역이다. 자세한 로드맵은 docs/05_DELIVERY_PLAN.md, 기술 방향은 docs/04_TECHNICAL_DESIGN.md를 따른다.

- 인증 / 로그인 (소셜 인증, 사용자 계정) — localStorage 단일 사용자 → 다중 사용자 확장
- 실시간 동기화 (WebSocket 등) — 현재 setInterval 시뮬레이터를 실제 실시간으로 대체
- 백엔드 / DB 영속화 — 채팅·게시판·응원·밈을 서버에 저장
- 파일 업로드 — 밈 이미지 URL 입력을 실제 이미지 파일 업로드로 확장

---

## 7. Success Criteria

- 사용자는 닉네임·팀 설정 → 채팅 참여의 핵심 기능을 1분 안에 사용할 수 있다.
- 3회차 종료 시 브라우저에서 기본 흐름(닉네임·팀 선택 → 전체/팀별 채팅 → 밈 게시)을 확인할 수 있다.
- 4회차에서 핵심 흐름을 Playwright 테스트로 검증할 수 있다.
- 최종 결과물은 배포 가능한 형태여야 한다.
