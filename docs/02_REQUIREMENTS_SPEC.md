# Requirements Spec — 패독 코리아 (Paddock Korea)

## 1. Actors

| Actor | Description |
|---|---|
| Primary User | 닉네임을 설정하고 채팅·밈 기능으로 소통하는 한국의 F1 팬 |
| Admin | 이번 MVP에서는 제외. F1 101·경기 일정·순위 데이터는 정적 데이터로 처리한다. |

---

## 2. Main Use Cases

### UC-001. 팀 선택 및 닉네임 설정 (온보딩)

- **Actor:** Primary User
- **Goal:** 선호 팀과 닉네임을 설정하여 팀 채팅방과 전체 채팅방에 참여할 수 있는 상태가 된다.
- **Precondition:** 사용자가 처음 앱에 진입했거나 설정 정보가 없는 상태이다.
- **Main Flow:**
  1. 사용자가 앱 첫 화면에서 닉네임을 입력한다.
  2. 2026 시즌 11개 팀(McLaren, Ferrari, Red Bull 등) 또는 가상 옵션("응원 팀 없음", "올팬") 중 하나를 선택한다.
  3. 설정 정보가 로컬에 저장되고, 메인 화면(전체 채팅)으로 이동한다.
- **Alternative Flow 1:** 이미 설정 정보가 저장된 사용자는 온보딩을 건너뛰고 메인 화면으로 진입한다.
- **Alternative Flow 2:** "응원 팀 없음"(`selectedTeamId="none"`) 또는 "올팬"(`selectedTeamId="all"`)을 선택해도 온보딩을 완료할 수 있다.
- **Result:** 닉네임·선택 팀(또는 가상 옵션)이 저장되며, 이후 채팅 메시지에 팀 색상과 닉네임이 표시된다.

---

### UC-002. 전체 채팅 참여 (Main Straight, Home 내)

- **Actor:** Primary User
- **Goal:** 팀에 상관없이 모든 팬과 실시간으로 소통한다.
- **Precondition:** 온보딩이 완료된 상태이다.
- **Main Flow:**
  1. 사용자가 Home 탭에 진입하면 우측(데스크톱)/하단(모바일)에 전체 실시간 채팅이 컴팩트하게 표시된다.
  2. 최근 채팅 메시지 목록이 시간순으로 표시되며, 시뮬레이터가 주기적으로 새 메시지를 추가한다.
  3. 사용자가 입력창에 메시지를 작성하고 전송 버튼(또는 Enter)을 누른다.
  4. 메시지가 채팅창에 즉시 표시되고 스크롤이 최신으로 이동한다.
- **Alternative Flow 1:** "전체화면" 버튼을 누르면 채팅이 오버레이로 확대되고, "닫기"로 복귀한다.
- **Alternative Flow 2:** 메시지 입력창이 비어 있으면 전송 버튼이 비활성화된다.
- **Result:** 메시지가 전체 채팅방에 등록되어 닉네임·팀 색상과 함께 표시된다.

> 채팅은 별도 탭이 아니라 **Home 대시보드 안에 통합**되어 있다. (기존 "The Main Straight" 별도 탭을 Home으로 흡수)

### UC-003. 팀 게시판 참여 (Board)

- **Actor:** Primary User
- **Goal:** 전체/팀별 게시판에서 글·댓글·좋아요로 소통하고, 다른 팀 게시판도 둘러본다.
- **Precondition:** 온보딩이 완료된 상태이다.
- **Main Flow:**
  1. 사용자가 Board 탭을 선택한다. 기본 스코프는 "전체"다.
  2. "팀별" 스코프로 전환하면 기본 팀(실제 팀 사용자는 자기 팀, "올팬"·"응원 팀 없음"은 첫 번째 팀) 게시판이 열린다.
  3. 글쓰기 버튼으로 제목·본문을 입력해 글을 게시하거나, 글에 좋아요·댓글을 남긴다.
  4. 글 목록은 최신순/인기순으로 정렬할 수 있다.
- **Alternative Flow 1:** 사용자는 팀 탭에서 다른 팀 게시판을 **읽기 전용**으로 열람할 수 있다. 이때 글쓰기·댓글 입력이 잠기고 "읽기 전용" 안내 배지가 표시된다.
- **Alternative Flow 2:** "올팬"(`selectedTeamIds=["all"]`) 사용자는 모든 팀 게시판에 글을 쓸 수 있다. "응원 팀 없음"(`selectedTeamIds=[]`) 사용자는 팀 게시판이 모두 읽기 전용이다.
- **Alternative Flow 3:** 제목 또는 본문이 비어 있으면 게시 버튼이 비활성화된다.
- **Result:** 작성 권한이 있는 게시판에 글·댓글이 등록되며, 권한이 없는 팀 게시판은 열람만 가능하다.

> 작성 권한 규칙: `canPostInTeamChat(profile, teamId)` — `selectedTeamIds`가 `teamId`를 포함하거나 `"all"`을 포함하면 작성 가능. (전체 스코프는 온보딩 완료 사용자 모두 작성 가능)

### UC-004. 밈 업로드 (Meme Box)

- **Actor:** Primary User
- **Goal:** 채팅과 별도의 밈 공유 공간에 이미지 URL과 캡션으로 밈을 게시한다.
- **Precondition:** 온보딩이 완료된 상태이다.
- **Main Flow:**
  1. 사용자가 Meme Box 탭에서 밈 올리기 버튼을 누른다.
  2. 이미지 URL과 캡션(선택)을 입력한다.
  3. 게시 버튼을 누른다.
  4. 밈이 Meme Box 피드 최상단에 표시된다.
- **Alternative Flow:** 이미지 URL이 비어 있거나 유효하지 않으면 게시 버튼이 비활성화되거나 오류 안내가 표시된다.
- **Result:** 밈이 Meme Box 피드에 등록된다.

---

### UC-005. 밈 조회 및 반응

- **Actor:** Primary User
- **Goal:** Meme Box에서 다른 팬이 올린 밈을 보고 좋아요로 반응한다.
- **Precondition:** Meme Box에 밈이 하나 이상 존재한다.
- **Main Flow:**
  1. 사용자가 Meme Box 탭을 열면 밈 피드가 최신순으로 표시된다.
  2. 사용자가 원하는 밈의 좋아요 버튼을 누른다.
  3. 해당 밈의 좋아요 수가 즉시 반영된다.
- **Alternative Flow:** 밈이 없으면 "아직 올라온 밈이 없습니다" 안내가 표시된다.
- **Result:** 좋아요 반응이 저장되고 수치가 업데이트된다.

---

### UC-006. F1 입문 가이드 조회 (F1 101)

- **Actor:** Primary User
- **Goal:** F1 용어 사전과 경기 진행 방식을 확인하여 F1을 처음 접하는 팬도 이해할 수 있다.
- **Precondition:** 없음.
- **Main Flow:**
  1. 사용자가 F1 101 탭을 선택한다.
  2. 카테고리 목록(용어 사전 / 경기 방식 / 레이스 위켄드 구조)이 표시된다.
  3. 항목을 선택하면 카드 형식의 상세 설명이 표시된다.
- **Alternative Flow:** 없음.
- **Result:** 사용자가 F1 기초 지식을 습득한다.

---

### UC-007. 경기 일정 확인

- **Actor:** Primary User
- **Goal:** 다가오는 레이스 위켄드 일정을 KST 기준으로 확인한다.
- **Precondition:** 없음.
- **Main Flow:**
  1. 사용자가 Pit Wall 탭의 경기 일정 섹션에 진입한다.
  2. 현재 시즌의 레이스 위켄드 목록이 KST 기준 날짜·시간과 함께 표시된다.
  3. 가장 가까운 다음 레이스가 상단에 강조 표시된다.
- **Alternative Flow:** 없음.
- **Result:** 사용자가 다음 경기 일정을 확인한다.

---

### UC-008. 챔피언십 순위 확인

- **Actor:** Primary User
- **Goal:** 현재 시즌 드라이버·컨스트럭터 챔피언십 순위를 확인한다.
- **Precondition:** 없음.
- **Main Flow:**
  1. 사용자가 Pit Wall 탭에서 드라이버 순위 또는 컨스트럭터 순위 탭을 선택한다.
  2. 순위, 이름, 포인트가 정렬된 목록으로 표시된다.
  3. 완료된 세션을 선택하면 세션별 결과(드라이버 헤드샷·순위·포인트)가 표시된다.
- **Alternative Flow:** OpenF1 호출 실패 시 정적 폴백 데이터로 순위·일정을 표시한다.
- **Result:** 사용자가 현재 시즌 순위·세션 결과를 파악한다.

---

### UC-009. 팀 응원 및 랭킹 (Cheer)

- **Actor:** Primary User
- **Goal:** 내 팀에 응원을 보내고 팀별 누적 응원 랭킹을 확인한다.
- **Precondition:** 온보딩이 완료된 상태이다.
- **Main Flow:**
  1. 사용자가 Home 화면에서 응원 패널을 본다.
  2. 응원 버튼을 누르면 선택 팀의 응원 수가 +1 되고, 랭킹에 즉시 반영된다.
  3. 팀별 누적 응원(시드 baseline + 사용자 누적)이 랭킹 목록으로 정렬되어 표시된다.
- **Alternative Flow 1:** 오늘 이미 응원했다면(KST 기준 하루 1회) 응원 버튼이 비활성화되고 "오늘 응원 완료" 상태가 된다.
- **Alternative Flow 2:** "응원 팀 없음" 사용자는 응원 대상 팀이 없으므로 패널에서 응원 대상을 선택하도록 안내한다.
- **Result:** 응원 수가 localStorage에 누적 저장되고 랭킹에 반영된다.

---

### UC-010. 프로필·테마 설정 (My Page)

- **Actor:** Primary User
- **Goal:** 닉네임·응원 팀을 변경하고, 화면 테마를 전환하며, 내가 쓴 글을 확인한다.
- **Precondition:** 온보딩이 완료된 상태이다.
- **Main Flow:**
  1. 사용자가 My Page 탭(또는 헤더 프로필)으로 진입한다.
  2. 닉네임(1–15자)과 응원 팀(최대 2개 또는 가상 옵션)을 수정한다.
  3. 변경 사항이 있을 때만 "변경 저장" 버튼이 활성화되고, 저장 시 localStorage·헤더에 즉시 반영된다.
  4. 라이트/다크 테마 토글로 화면 테마를 전환한다.
  5. 내가 작성한 게시글 목록(최신순)을 확인한다.
- **Alternative Flow:** 프로필이 없으면(온보딩 미완료) 온보딩을 먼저 완료하도록 안내한다.
- **Result:** 프로필·테마가 저장되고 앱 전체에 반영된다.

## 4. Non-functional Requirements

| ID | Requirement |
|---|---|
| NFR-001 | 모바일 화면(375px 이상)에서도 핵심 기능을 사용할 수 있어야 한다. |
| NFR-002 | 버튼과 입력 필드는 접근 가능한 이름(aria-label 또는 label)을 가져야 한다. |
| NFR-003 | 민감한 정보(API Key 등)는 GitHub 저장소에 커밋하지 않는다. |
| NFR-004 | 로그인 없이 닉네임·팀 정보를 로컬 스토리지에 저장하여 단일 사용자 기준으로 구현한다. |
| NFR-005 | 페이지 초기 로딩은 3초 이내여야 한다. |
| NFR-006 | 서비스 UI는 한국어를 기본 언어로 사용한다. |

## 5. Acceptance Criteria

### AC-001. 팀 선택 및 닉네임 설정

Given 사용자가 처음 앱에 진입하여 설정 정보가 없는 상태일 때  
When 닉네임을 입력하고 팀(11개 실제 팀 또는 "응원 팀 없음"·"올팬" 가상 옵션)을 선택하면  
Then 설정이 저장되고 메인 채팅 화면으로 이동한다.

> 가상 옵션을 선택해도 완료 버튼이 활성화되며 온보딩이 완료된다.

---

### AC-002. 전체 채팅 메시지 전송

Given 온보딩이 완료된 사용자가 Home 탭의 전체 채팅에 있을 때  
When 메시지를 입력하고 전송 버튼(또는 Enter)을 누르면  
Then 해당 메시지가 닉네임·팀 색상과 함께 채팅창에 즉시 표시되고 스크롤이 최신으로 이동한다.

---

### AC-003. 팀 게시판 글 작성

Given 온보딩이 완료된 사용자가 Board의 자기 팀 게시판(또는 "올팬"으로 임의 팀 게시판)에 있을 때  
When 제목·본문을 입력하고 게시 버튼을 누르면  
Then 해당 글이 그 팀 게시판에만 등록되고 피드 최상단에 표시된다.

> 작성 권한이 없는 다른 팀 게시판(타팀 팬 또는 "응원 팀 없음" 사용자)에서는 글쓰기·댓글이 잠기고 "읽기 전용" 배지가 표시되며, 글 목록은 열람할 수 있다.

---

### AC-004. 밈 업로드

Given 사용자가 유효한 이미지 URL을 입력했을 때  
When 게시 버튼을 누르면  
Then 해당 밈이 Meme Box 피드 최상단에 표시된다.

---

### AC-005. 밈 좋아요

Given Meme Box에 밈이 하나 이상 존재할 때  
When 사용자가 좋아요 버튼을 누르면  
Then 해당 밈의 좋아요 수가 1 증가하여 즉시 표시된다.

---

### AC-006. F1 101 가이드 조회

Given 사용자가 F1 101 탭에 진입했을 때  
When 카테고리 항목(용어 사전 / 경기 방식 등)을 선택하면  
Then 해당 항목의 상세 설명이 카드 형식으로 표시된다.

---

### AC-007. 경기 일정 확인

Given 사용자가 경기 일정 화면에 진입했을 때  
When 화면이 로드되면  
Then 다가오는 레이스 위켄드 일정이 KST 기준 날짜·시간과 함께 표시되며, 가장 가까운 일정이 상단에 강조된다.

---

### AC-008. 챔피언십 순위 확인

Given 사용자가 Pit Wall 탭에 진입했을 때  
When 드라이버 또는 컨스트럭터 탭을 선택하면  
Then 현재 시즌 순위, 이름, 포인트가 정렬된 목록으로 표시된다.

---

### AC-009. 팀 응원

Given 온보딩이 완료된 사용자가 Home의 응원 패널에서 오늘 아직 응원하지 않았을 때  
When 응원 버튼을 누르면  
Then 선택 팀의 응원 수가 1 증가하여 랭킹에 즉시 반영되고, 응원 버튼은 "오늘 응원 완료" 상태로 비활성화된다.

---

### AC-010. 프로필 변경

Given 사용자가 My Page에서 닉네임 또는 응원 팀(최대 2개)을 수정했을 때  
When "변경 저장" 버튼을 누르면  
Then 변경 사항이 localStorage에 저장되고 헤더 프로필에 즉시 반영된다.

---

### AC-011. 테마 전환

Given 사용자가 라이트/다크 테마 토글을 사용할 때  
When 테마를 전환하면  
Then 화면 테마가 즉시 바뀌고, 재방문 시에도 설정이 유지된다.
